# p3-dashboard setup

쿠버네티스의 웹기반 대시보드 및 설치형 대시보드인 OpenLens를 사용한다.



### 1. 대시보드 배포

* dashboard-recommended.yml 파일을 아래와 같이 생성
  * k8s 권장 설정에서 서비스 유형을 NodePort로 변경하고 포트를 30443으로 변경함
  * 네임스페이스는 `kubernetes-dashboard` 사용
* `kubectl apply -f dashboard-recommended.yml`으로 배포한다.

### 2. 사용자 생성

* `admin-user`를 생성하면서 ServiceAccount, ClusterRoleBinding을 설정한다.
* 아래 내용을 `dashboard-admin-user.yml` 파일로 저장하고, `kubectl apply -f dashboard-admin-user.yml`로 생성한다.

```yaml
apiVersion: v1
kind: ServiceAccount
metadata:
  name: admin-user
  namespace: kubernetes-dashboard
---
apiVersion: rbac.authorization.k8s.io/v1
kind: ClusterRoleBinding
metadata:
  name: admin-user
roleRef:
  apiGroup: rbac.authorization.k8s.io
  kind: ClusterRole
  name: cluster-admin
subjects:
- kind: ServiceAccount
  name: admin-user
  namespace: kubernetes-dashboard
```

* admin-user에 대한 로그인 토큰을 생성하고 k8s 사용자의 kube config에 추가한다.

```bash
# admin-user 로그인 토큰 생성
token=$(kubectl -n kubernetes-dashboard create token admin-user)

# admin-user 토큰을 kube config 파일에 추가
echo "    token: $token" >> ~/.kube/config
```



### 3. 웹 대시보드 접속

브라우저에서 `https://192.168.0.10:30443/`으로 접속하다.

바로 앞 단계에서 생성한 토큰을 입력하면 로그인이 가능하다.

>(문제해결) 연결이 비공개로 설정되어 있지 않습니다.
>
>유효한 TLS 인증서를 사용하지 않아 위와 같은 오류 메시지가 나타날 것이다. 브라우즈 화면을 클릭 후 `thisisunsafe`를 입력하면 접속이 가능해진다.



### 4. OpenLens 대시보드 접속

OpenLens에 Cluster를 추가하면서   ` ~/.kube/config`의 내용을 붙여 넣으면 대시보드를 이용할 수 있다.

(참고) OpenLens 메뉴: Catalog > Categories > Clusters > (+) Add from kubeconfig
